
# 데이터 전처리 전략 리포트

## 1. 개요
Boosting 모델용 전처리. 불필요한 파생변수 없이 깔끔하게 처리함.

## 2. 삭제한 컬럼
- `ID`: 식별자. 학습에 무관.
- `generation`: 기수. Train=9기, Test=10기로 나뉘어 있어서 모델이 이걸 보고 치팅할 수 있음.

## 3. 결측치 처리
- **수치형**: `-1`로 채움. Boosting 모델은 결측을 자체 처리 가능하지만, 명시적으로 "이건 결측이었다"를 표시해주는 게 안전함.
- **범주형**: `Unknown` 문자열로 채움. 결측 자체가 하나의 범주가 됨.

## 4. 인코딩 전략

### A. One-Hot Encoding (20개 컬럼)
고유값이 15개 미만인 컬럼에 적용.
- 예: `major_data` (True/False), `nationality` (내국인/외국인) 등
- 결과: 각 범주별로 0/1 컬럼이 생성됨.

### B. Label Encoding (13개 컬럼)
고유값이 15개 이상인 컬럼에 적용.
- 예: `school1` (학교 수십 개), `major1_1` (전공 수십 개), `whyBDA` (선택지 여러 개) 등
- 결과: 각 범주를 0, 1, 2, 3... 숫자로 변환.

## 5. 최종 결과
- **Train**: 748행, 157열
- **Test**: 814행, 156열 (Target 컬럼 제외)

## 6. 이전 버전과 차이
- ~~텍스트 길이 파생변수~~ 삭제. 해당 컬럼들이 체크박스 응답이라 길이가 의미 없음.
- 결측치 처리: Median → `-1`로 변경.
